# AGENT WORK MODE

1. Работать инкрементально, без массовых переписываний.
2. Перед изменениями кратко писать план (2–5 строк).
3. Не удалять существующую логику без явного запроса.
4. Не объединять версии файлов без команды.
5. Один логический шаг = один коммит.
6. После изменений показывать:
   - git diff --stat 
   - список изменённых файлов
7. Не задавать лишних подтверждений, если задача чёткая.
8. Всегда учитывать:
   - ROADMAP.md 
   - STATUS.md 
   - VISION.md 
   - текущую архитектуру server/app
